✅ Prérequis
Avant de commencer, assurez-vous d'avoir :

Node.js (version 16 ou supérieure, recommandé v20.x)

npm ou yarn (gestionnaires de paquets)

Un éditeur de code comme Visual Studio Code

📚 Guide Git & GitHub
🛠️ Configuration initiale de Git
bash
Copy
Edit
# Vérifier que Git est installé
git --version

# Configurer votre identité (à faire une seule fois)
git config --global user.name "Votre Nom"
git config --global user.email "votre.email@example.com"
🌿 Créer et Travailler sur une Branche
bash
Copy
Edit
# Créer une nouvelle branche
git checkout -b nom-de-la-branche

# Voir toutes les branches
git branch

# Changer de branche
git checkout nom-de-la-branche
📦 Sauvegarder ses modifications (Commit)
bash
Copy
Edit
# Vérifier les changements
git status

# Ajouter tous les fichiers modifiés
git add .

# OU ajouter un seul fichier
git add chemin/vers/le/fichier

# Créer un commit avec un message
git commit -m "Votre message de commit clair et précis"
🔄 Envoyer son travail sur GitHub (Push)
bash
Copy
Edit
# Envoyer votre branche vers GitHub
git push origin nom-de-la-branche

# Si c'est une nouvelle branche
git push --set-upstream origin nom-de-la-branche
🚀 Créer une Pull Request (PR)
Allez sur GitHub.

Ouvrez votre projet.

Cliquez sur "Compare & Pull Request".

Remplissez un titre et une description claire.

Cliquez sur "Create Pull Request".

🔀 Merger une Pull Request
Une fois la PR validée, cliquez sur "Merge Pull Request".

Puis cliquez sur "Delete Branch" pour supprimer la branche fusionnée.

📥 Mettre à jour votre projet local (Pull)
bash
Copy
Edit
# Depuis la branche principale
git checkout main

# Récupérer les dernières modifications
git pull origin main
📝 Bonnes Pratiques Git & GitHub
Créez une branche par fonctionnalité (feature/), correction (fix/) ou amélioration (improvement/).

Rédigez des commits courts et significatifs.

Synchronisez régulièrement votre travail (git pull) pour éviter les conflits.

Décrivez clairement vos Pull Requests.

📄 Résumé des Commandes Git importantes

Action	Commande
Initialiser un projet Git	git init
Cloner un projet existant	git clone URL
Créer une branche	git checkout -b nom-branche
Voir les branches	git branch
Changer de branche	git checkout nom-branche
Ajouter les fichiers	git add .
Commiter les changements	git commit -m "Message"
Pousser vers GitHub	git push origin nom-branche
Récupérer les mises à jour	git pull origin main
